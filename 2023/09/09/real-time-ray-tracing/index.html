Real-time Ray Tracing

发表于2023-09-09|更新于2023-09-09|GAMES202

Currently, in the field of real-time ray tracing, we sample once per pixel, which inevitably leads to a large amount of noise.

So the key technique is denoising. Here are several methods of denoising to introduce.

Temporal

Suppose the previous frame is denoised and reuse it. Use motion vector to find previous locations.

For a pixel in current frame, we find its corresponding point in world space by G-buffer. If there’s no G-buffer, use transformation. Then we find its corresponding pixel in the previous frame using transformation.

The motion vector is a vector from the current pixel to the corresponding pixel in the previous frame.

This method can be expressed by the following equation. Here, Cˉi\bar C_iCˉi​ denotes the denoised pixel value in the current frame, C~i\tilde C_iC~i​ denotes the original pixel value in the current frame, Cˉi−1\bar C_{i-1}Cˉi−1​ denotes the denoised pixel value in the previous frame, F\mathrm FF is a spatial filter.

Cˉi=α F(C~i)+(1−α) Cˉi−1\bar C_i=\alpha\,\mathrm{F}(\tilde C_i)+(1-\alpha)\,\bar C_{i-1} Cˉi​=αF(C~i​)+(1−α)Cˉi−1​

Usually α\alphaα is set between 0.1 and 0.2, meaning that 80%-90% contributions are from last frames. The effect of this method is obvious.

But temporal accumulation is invalid sometimes.

  • switching scene
  • walking backwards in a hallway
  • suddenly appearing background

In a word, the point in the previous frame may be occluded by other objects, or out of screen.

Taking the above figure as an example, the left is the previous frame and the right is the current frame. The corresponding pixel in the previous frame actually is on the yellow box, not on the white object. Some adjustments is necessary to alleviate occlusion.

One method is clamping. Clamp previous toward current. Here’s how it would work. In the current frame, we get the mean μ\muμ and standard deviation σ\sigmaσ within a region around the pixel. If the previous pixel value beyonds the range (μ−kσ,μ+kσ)(\mu-k\sigma,\mu+k\sigma)(μ−kσ,μ+kσ), clamp it.

The other is using object ID to detect temporal failure. If the corresponding pixel in the previous frame doesn’t correspond to the same object, set the α\alphaα larger, or 1, to reduce the contribution from the previous frame.

Nonetheless both two methods re-introduce noise. And they can’t solve the problems of shading, such as glossy reflection and shadows in the dynamic scenes, as the following figure shown.

Spatial

Joint Bilateral Filtering

Before introducing joint bilateral filtering, introduce bilateral filtering first.

Gaussian filtering blurs the whole image, including boundaries, but boundaries are the high frequency information that we want to keep. How to keep the boundaries?

When XXX and YYY are independent, the means of XXX and YYY are both 0, and the variances of XXX and YYY are the same, the 2-D Gaussian distribution is

f(x,y)=12πσ2exp(−x2+y22σ2)f(x,y)=\frac{1}{2\pi\sigma^2}\mathrm{exp} \left(-\frac{x^2+y^2}{2\sigma^2}\right) f(x,y)=2πσ21​exp(−2σ2x2+y2​)

Previously, we use Gaussian filtering based on the distance of pixels. xxx and yyy are denote the distances from a pixel to the current pixel on the X-axis and Y-axis in the screen respectively.

A filter can be considered as a weight matrix. Now let’s construct the weight matrix base on the distance and color both. Multiply two Gaussian filter.

w(i,j,k,l)=12πσd212πσr2⋅exp(−(i−k)2+(j−l)22σd2−∥I(i,j)−I(k,l)∥22σr2)\begin{aligned} w(i,j,k,l)=\frac{1}{2\pi\sigma_d^2}\frac{1}{2\pi\sigma_r^2}\cdot\newline \mathrm{exp}\left(-\frac{(i-k)^2+(j-l)^2}{2\sigma_d^2}-\frac{\Vert I(i,j)-I(k,l)\Vert^2}{2\sigma_r^2}\right) \end{aligned} w(i,j,k,l)=2πσd2​1​2πσr2​1​⋅exp(−2σd2​(i−k)2+(j−l)2​−2σr2​∥I(i,j)−I(k,l)∥2​)​

where the current pixel is (i,j)(i,j)(i,j), the other pixel is (k,l)(k,l)(k,l), and I(i,j)I(i,j)I(i,j) and I(k,l)I(k,l)I(k,l) are the color value of the pixels. This is intuitive because areas with significant color changes are considered boundaries.

Ordinary Gaussian filtering has 1 metrics, the distance. Bilateral filtering has 2 metrics, the distance and the color. Can we use more metrics to better guide filtering?

Yes, that is joint bilateral filtering. Apart from the distance and the color, we can use depths, normals, object IDs, and so on. These information can be stored in G-buffer from which accessing information is almost free.

Taking the following figure as an example, why we do not blur the boundary using joint bilateral filtering?

  • A and B: depth
  • B and C: normal
  • D and E: color

What’s more, Gaussian filter is not the only choice. Any function that decreases with “distance” would work.

Implementing Large Filters

Recall that for each pixel, we need to loop over all its N×NN\times NN×N neighborhood. For large filters, this can be prohibitively heavy. Here are two different solutions to large filters.

Separate Passes

This method is specific to Gaussian filtering. Separate it into a horizontal pass (1×N1\times N1×N) and a vertical pass (N×1N\times 1N×1). Computational complexity changes from O(N2)O(N^2)O(N2) to O(N)O(N)O(N).

The following section theoretically explains its rationality.

Outlier Removal

Filtering is not almighty. Sometimes the filtered results are still noisy, even blocky. Mostly due to extremely bright pixels called outlier. Because we use the image with high dynamic range before outputting to the screen, values of these outliers may be very large.

We should remove these outliers before filtering.

For each pixel, take a look at its N×NN\times NN×N neighborhood. Compute mean μ\muμ and variance σ2\sigma^2σ2 of this region. These with values outside the interval [μ−kσ,μ+kσ][\mu-k\sigma,\mu+k\sigma][μ−kσ,μ+kσ] are outliers.

To remove these outliers, clamp any value outside [μ−kσ,μ+kσ][\mu-k\sigma,\mu+k\sigma][μ−kσ,μ+kσ] to this range.

Specific Filtering Solutions

Spatiotemporal Variance-Guided Filtering

SVGF is very similar to the basic spatio-temporal denoising scheme, but with some additional variance analysis and tricks.

In the joint bilateral filtering, there are 3 factors to guide filtering, those are depths, normals, and colors. We simply used the difference between them as the metrics.

However, in the method of SVGF we use more reasonable metrics to weight.

For depths, A and B in the following figure are on the same plane, so they should contribute greatly to each other. But their depths have a big gap. If we simply use their difference as the metrics, they will contribute to each other too little.

So people propose the following equation as the weight, which considers not only the difference of depth but also the gradient of depth.

wz=exp(−∣z(p)−z(q)∣σz∣∇z(p)⋅(p−q)∣+ε)w_z=\mathrm{exp}\left(-\frac{|\mathrm z(\mathbf p)-\mathrm z(\mathbf q)|}{\sigma_z|\nabla \mathrm z(\mathbf p)\cdot(\mathbf p-\mathbf q)|+\varepsilon}\right) wz​=exp(−σz​∣∇z(p)⋅(p−q)∣+ε∣z(p)−z(q)∣​)

where zzz denotes the depth field, ppp and qqq denote pixels, σz\sigma_zσz​ is a adjustable parameter, ε\varepsilonε prevents denominator from being zero.

For normals, the weights can be written as
