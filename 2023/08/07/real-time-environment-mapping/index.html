Let’s focus on how to compute the first term of the approximation of rendering equation above

∫ΩfrL(p,ωi)dωi∫Ωfrdωi\frac{\int_{\Omega_{f_r}} L(p,\omega_i)\mathrm{d}\omega_i}{\int_{\Omega_{f_r}} \mathrm{d}\omega_i} ∫Ωfr​​​dωi​∫Ωfr​​​L(p,ωi​)dωi​​

As the figure shown, if the material of surface is glossy, the incident rays around a particular direction contribute most of the irradiance. The rest of the rays contribute little, so we ignore them. These rays that contribute a lot to the shading point are concentrated in a solid Angle, whose degree varies with the material. This solid angle corresponds to a region of the environment map. We just need to take the incident light from this region into account. This process is equivalent to that we intersect the environment map at a point in the direction of the perfect specular reflection and take the region centered around the intersection point as light source. The size of the region varies with the material naturally too.

Understanding the above process, we can pre-filter the environment map and generate a set of differently filtered environment map. The solide angel corresponds to the size of the filter. At run time, we read the appropriate value from the appropriate filtered environment map. Filter size in-between can be approximated via trilinear interpolation.

Preliminary

In mathematics, a basis function is an element of a specific basis in the Function space. Every Continuous function in the Function space can be expressed as a linear combination of basis functions, just as every vector in the Vector space can be expressed as a linear combination of basis vectors.

For example, Fourier expansion is written as

f(t)=a0+∑n=1∞(ancos⁡nω0t+bnsin⁡nω0t)f(t)=a_0+\sum_{n=1}^{\infty}(a_n\cos{n\omega_0t}+b_n\sin{n\omega_0t}) f(t)=a0​+n=1∑∞​(an​cosnω0​t+bn​sinnω0​t)

Here, cos⁡nω0t\cos n\omega_0 tcosnω0​t and sin⁡nω0t\sin n\omega_0 tsinnω0​t are basic functions. f(t)f(t)f(t) can be expressed as linear combination of them.

Two-dimensional function can express a surface in three-dimensional space. It can also be expressed as linear combination of basic functions.

Spherical harmonics is a set of basis functions that can express a two-dimensional function.

Look at the following figure. There is l=nl=nl=n written on the left side of the n-th row, which represents the n-th order spherical harmonics. We find that the n-th order spherical harmonics contains 2n+12n+12n+1 functions and the first n orders spherical harmonics contains 2n2^n2n functions. The higher the order, the higher the frequency.

We need to pre-compute every tijt_{ij}tij​ and store them in a matrix because they have two dimension. And we still need to pre-compute every lil_ili​ and store them in a vector.

In order to express a function accurately, we use the linear combination of infinite spherical harmonics. In engineering practice, we only use finite spherical harmonics as basis functions.

In engineering practice, we often choose the first 9, 16 or 25 spherical harmonic functions, in other words, the first 3, 4 or 5 orders of spherical harmonics. The more functions, the more accurate the result, but the higher the overhead. We choose the first several orders, so that we lost some high frequency information.

Note that PRT is limited to rendering static scenes. Once the objects in the scene move, the occlusion relationship will change, and the pre-computed information will be invalid.

Despite this, the light source (environment map) can rotate, because the rotated lighting can easily expressed by rotating its basis functions. And a rotated basis function is no longer a basis function but can easily expressed by the basis functions in its same order.

The PRT can take the indirect illumination into account. It can generate shadows formed by multiple reflections of light in the pre-computation stage. Runtime is independent to the complexity of transport paths.
