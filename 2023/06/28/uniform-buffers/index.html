笔记：Uniform Buffers

发表于2023-06-28|更新于2023-07-17|Vulkan

Descriptor layout and buffer

Shader可以通过descriptor获取资源（buffer和image）。Descriptor的用法如下：

  • 在pipeline的创建期间指定一个descriptor layout
  • 从一个descriptor pool中分配一个descriptor set
  • 在渲染过程中绑定descriptor set

Descriptor Layout规定了资源的类型；Descriptor Set指定了实际的image或buffer资源。

Alignment requirements

Vulkan期待接收的数据是对齐的。例如

  • Scalars have to be aligned by N (= 4 bytes given 32 bit floats).
  • A vec2 must be aligned by 2N (= 8 bytes)
  • A vec3 or vec4 must be aligned by 4N (= 16 bytes)
  • A nested structure must be aligned by the base alignment of its members rounded up to a multiple of 16.
  • A mat4 matrix must have the same alignment as a vec4.

在C++11标准中， 有alignas关键字用来对齐数据。

前几小节中的代码没有显式地对齐，也能正常运行，这是因为侥幸，结构体中的数据恰好本来就是对齐的。

总结

Vulkan新手的一些理解，可能有误。

一个pipleline拥有p个descriptor set layout，也就是说可以绑定p个descriptor set。一个descriptor set包含q个bingding。一个binding包含r个descriptor。

在shader这端，以GLSL语言为例，

这段代码所声明的变量就是第0号descriptor set layout处的descriptor set的第0号binding的若干个descriptor。

Descriptor像连接pipeline和数据资源的接口，好比是数据线。在创建pipeline时，我们为pipeline创建了n个descriptor set layout，好比为设备A制造了n个插槽，例如type-C形制的插槽。然后我们按照这些descriptor set layout规定的样式，创建descriptor set，好比我们必须按照type-C的插槽的样式来制造数据线的插头，要不然你制造了一个雷电插头，根本就插不进去。制造完数据线后，我们还要把存储数据的设备连接在数据线的另一头，设备A才能通过数据线获取数据，这个过程就是我们上文中调用vkUpdateDescriptorSets的过程。
